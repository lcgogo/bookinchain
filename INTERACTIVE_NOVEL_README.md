# InteractiveNovelDAO 功能说明

## 核心概念

这个合约实现了 **"互动小说 + DAO治理 + 衍生授权"** 三位一体的系统。

## 工作流程

### 1. 原作者发布章节
```
作者 → 发布章节 + 多个分支选项 → 开始投票期
```
- 每个章节至少有2个分支剧情选项
- 投票期结束后，票数最多的分支成为主线

### 2. 读者投票决定走向
```
读者 → 支付投票费用 → 选择喜欢的分支
```
- 每人每章节只能投一次
- 可以基于票数或投票金额统计（当前用票数）

### 3. 分叉授权（衍生创作）
```
创作者 → 支付授权费 → 从某章节开始新故事线 → 继续创作
```

## 关键参数

| 参数 | 说明 |
|------|------|
| `baseChapterFee` | 基础章节阅读费用 |
| `voteFee` | 每次投票的费用 |
| `forkLicenseRate` | 分叉授权费率（影响阅读量系数） |

## 分叉授权费用计算

```
授权费 = (前置章节数 × 单章费用) × 阅读量系数
阅读量系数 = 1 + (总阅读人数 × 费率 / 1000)
```

**示例**：
- 假设单章费用 0.01 ETH
- 从第10章开始分叉，前置章节有9个
- 故事线有500个读者，费率5000基点
- 计算：(9 × 0.01) × (1 + 500 × 5000/1000) = 0.09 × 3.5 = 0.315 ETH

## 使用场景

### 场景1：主线创作
1. 作者发布第1章，提供2个分支
2. 读者投票选择A分支
3. 作者根据A分支写第2章

### 场景2：衍生创作
1. 读者喜欢B分支但未胜出
2. 支付授权费后，从第1章B分支开始创作新故事线
3. 成为该故事线的"作者"，继续发布章节

## 收益分配

| 来源 | 收益方 |
|------|--------|
| 章节阅读费 | 原作者 |
| 投票费用 | 原作者 |
| 分叉授权费 | 原作者 |

## 扩展建议

1. **版税分成**：给参与投票/评论的读者代币奖励
2. **NFT化**：每章铸造NFT，允许二级市场交易
3. **收益分成**：衍生故事线收益与原作者分账
4. **时间锁**：分叉授权需要等待投票结束后才能申请
